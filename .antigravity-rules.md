# Reglas de Agente para KontrolSage

Este proyecto es una aplicación de escritorio **multiplataforma** (Windows, macOS, Linux) usando **Avalonia UI** con .NET 8 y C#. Además, implementa el patrón arquitectónico **MVVM** (Model-View-ViewModel) y utiliza **MongoDB** localmente como base de datos de gestión de proyectos.

**Regla de Oro Multiplataforma**: Todas las bibliotecas (`NuGet`), llamadas al sistema de archivos (`Path.Combine`, `Directory.SeparatorChar`), y herramientas (`FilePicker`, etc.) deben ser compatibles con los tres sistemas operativos principales. Queda estrictamente prohibido el uso de APIs dependientes de un solo sistema (ej. `Microsoft.Office.Interop`, `WinForms`, `WPF` dependencies, o rutas hardcodeadas como `C:\`).

## Convenciones de Arquitectura y Código

1.  **MVVM Estricto**: 
    - Toda la lógica de negocio, cálculos (ej. variables de EVM) y de presentación deben residir en los componentes `ViewModels`.
    - Las `Views` (`.axaml` y `.axaml.cs`) no deben contener lógica de la aplicación. El _code-behind_ se reserva estrictamente para interactuar con eventos inmediatos de la UI a bajo nivel solo si es estrictamente inevitable. Prefiera siempre usar *Data Binding* o comandos implementados en los ViewModels.

2.  **Base de Datos (MongoDB)**:
    - La configuración principal y cadena de conexión se halla en `DatabaseConfig.cs`. 
    - Las entidades o clases modelo de MongoDB (`Models/`) deben definir explícitamente atributos (`[BsonId]`, `[BsonRepresentation(BsonType.ObjectId)]`).
    - Las llamadas para interactuar con la Base de Datos (`CRUD`) deben aislarse y abstraerse dentro de un archivo de Servicio en el directorio `Services/` (por ejemplo, `EDTService.cs`). Todo acceso a datos en los ViewModels se realiza inyectando o instanciando los servicios y **nunca** directamente.

3.  **Avalonia UI**:
    - Use `x:DataType` en las vistas para aprovechar la verificación de bindings en tiempo de compilación.
    - Para elementos visuales constantes como Botones o TextBoxes, reutilizar estilos globales (*Styles*) en la Vista o aplicación.

4.  **Inyección de Dependencias (DI)**:
    - Evite instancias directas fuertemente acopladas entre ViewModels donde sea posible si hay contenedor de inversión de control (IoC). Si la aplicación lo declara en `App.axaml.cs`, registre los nuevos ViewModels o Servicios allí.

## Estilo General
- Mantenga la coherencia con el idioma usado para las variables, métodos o clases en el proyecto existente (ya que existen mezclas de español como `EDTService`, `AvanceService`, `costoRealService`). 
- Maneje todas las posibles excepciones o *Warnings* arrojados por el compilador, resolviendo posibles advertencias de referencias nulas (`CS8618`).
